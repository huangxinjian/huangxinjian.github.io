# MySQL 索引

什么是索引？索引是一种能够加快数据查询效率的数据结构，说通俗些，索引就相当于是字典上的目录，通过目录可以快速定位到对应的单词。

# 索引的好处和坏处

虽然说索引可以加快数据查询的速度，但索引并不是说越多越好，因为索引也是会占据存储空间的，如果盲目的建立索引，会导致数据文件变得很大。

而且，索引虽然能加快查询速度，但是会降低增删改的速度，因为每个索引都是维护了一颗索引树 B+ Tree ，每当有新的数据插入时，所有的索引树都必须插入这条新的数据，导致得重新调整树的结构。

# 索引的种类

MySQL 中索引有许多种，并且这些索引的都是属于存储引擎的，不同的存储引擎的索引是不一样的，例如：

  • 单列索引
  • 组合索引
  • 全文索引
  • 空间索引
  • 位图索引（Oracle）

# 单列索引

单列索引共有三种：

  • 唯一索引
  • 全文索引
  • 普通索引

# 普通索引

普通索引的创建方式如下：

CREATE INDEX index_name ON table(column(length)) ;
ALTER TABLE table_name ADD INDEX index_name (column(length)) ;

# 非聚集索引（MyISAM）

非聚集索引是 MyISAM 中的索引特点，其含义是，索引树下存储的值并不是数据记录的值，而是数据记录的地址，例如:

因此查询时，需要先通过索引树找到对应索引，然后再找到索引对应的数据的地址值，最后再根据地址值找到对应的数据内容。其中索引树是文件 .myi 的内容，右边的数据表是文件 .myd 的内容。

# 聚集索引（InnoDB）

聚集索引是 InnoDB 中的索引特点，其主键索引树下存储的值就是完整的数据内容，辅助索引树下存储的值是主键的值。

# 主键索引

主键索引图如下：

当建表指定主键时，就会生成这样一颗主键索引树，如果在建表时没有指定主键，就会先寻找唯一索引，将其作为主键，如果没有唯一索引，则会生成一个 rowid 伪列作为主键。

# 主键的选型

因为主键索引树是一定会存在的，因此主键的选型就变得重要，因为它会影响数据文件的大小。

首先不推荐 uuid 作为主键，因为它太大了，会占据很大的空间，比较推荐使用自增主键，这样不需要在每次新增数据时，都去比较主键的大小，而是直接追加即可，可以大大加快插入的速度。

对于分布式系统来说，主键 id 需要通过雪花算法来进行计算，否则会重复。

# 页分裂

当在插入数据的时候，如果不是自增主键的话，需要先判断出索引的位置在哪，如果刚好要插入的这一数据页的空间满了， 根据 B+ 树的算法，此时需要新增一个数据页，并且进行数据的移动，这个过程就称为页分裂。

因此这种情况就会影响到 mysql 的性能，并且还会降低空间的利用率，原本放在一个页的数据，现在分到两个页中，整体空间利用率降低大约 50%。

TIP

MySQL 默认一个数据页是 16K 大小，这个指标可以通过改变 innodb_page_size 选项对默认大小进行修改。

# 回表

首先看这么一条 SQL 语句, 其中 name 是普通索引，id 是主键索引

select * from t where name='Alice'
1

当指定条件 name=Alice 时，会去查辅助索引树，然后得到主键的 ID，接着再去主键索引树上进行查找，得到结果，这个过程就称为回表。

当发生回表时，需要查询两颗索引树，这样也会影响性能的，这时候就可以通过使用覆盖索引来进行优化了。

# 覆盖索引

覆盖索引，其实就是避免回表操作，尽量在一颗索引树上就得到我们想要的结果。例如上面的 SQL ，如果我们只需要查询 id ，就不用回表了，这样也算使用到了覆盖索引：

select id from t where name='Alice'
