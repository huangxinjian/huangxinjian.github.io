# 异步流程基本用法

javascript 的异步发展史是十分有趣且坎坷的，其中出现了很多的解决方案，但是我们这里只会讲最重要最实用的两种方案：

  • Promise
  • Async-Await

# 回调地狱

        }

# Promise

Promise 是 CommonJS 中提出来的一种回调地狱的解决办法，目前已经被 JavaScript 官方收录，因此也可以在 Chrome 中使用。

那什么是 Promise 呢？ Promise 其实就像是一个渣男，它向你承诺你目前的请求我不能马上给到你答复，但是未来我会给你答复。😄 这听起来就跟渣男说的差不多。

另外，Promise 还是一个状态机，它拥有三种状态：

  • Pending 初始化状态
  • Resolve 成功状态
  • Reject 异常状态

# Promise 的使用方法

Promise 的使用方法十分简单，通过 Promise 对象的构造函数，传入一个匿名 Function 回调，参数是 resolve 和 reject，如果成功执行了resolve,那么就会将resolve的值传给最近的then函数，如果出错执行了reject那么就交给catch来捕获异常。

例如：

// 定义
var promise = new Promise(function(resolve, reject){
  if(/*everything turned out fine*/) {
    resolve("Stuff worked")
  }else {
    reject(Error("It broke"))
  }
})

// 使用
promise.then(function(res){
  console.log(res)
  return Promise.reject(new Error("故意抛出错误"))
}).catch(function(err) {
  console.log(err)
})

# Promise 与 Callback 的区别

前面讲到，Promise 是为了解决回调地狱的，下面我们来看一下 Promise 如何解决回调地狱：

// Promise写法
function interview(round) {
  return new Promise((resolve, reject) => {
    if (Math.random() > 0.5) {
      resolve('success')
    } else {
      let err = new Error('cry')
      err.round = round
      reject(err)
    }
  })
}

interview(1)
  .then((res) => {
    // 再次异步
    return interview(2)
  })
  .then((res) => {
    // 再次异步
    return interview(3)
  })
  .then(res => {
    console.log(res)
  })
  .catch((err) => {
    console.log('cry at', err.round, 'round')
  })

可以看到，通过改写了回调地狱例子后的代码，看起来层次感很清晰。

这是因为 Promise 其实在异步调用外层包了两层，一层Promise，一层函数，然后把异步调用的结果包裹在Promise的状态中，然后把Promise作为结果返回，因此可以实现

  • 异步调用的代码在函数的定义当中
  • 异步结果的处理放在了函数外面的then函数或catch函数中

从写法上确实根本性的让两者分离。所以Promise的核心就是：将回调函数中的结果延时到then函数里处理或者交给catch全局异常处理

另外对于异常处理，如果我们在异步调用的回调函数中抛出异常，是无法被 try-catch 捕获到的，原因是大家处于不同的 call stack ，只有同步调用抛出的异常才能被捕获到。如果我们在异步调用的回调中抛出异常，会抛到浏览器或NodeJS全局中，从而使程序挂掉。

# 处理并行任务

对于多个并行的异步任务，如果我们需要它们同时成功或者同时失败的话，需要使用 Promise 提供的另外两个方法：

  • Promise.all
  • Promise.race

这里演示 Promise.all 的用法：




































 
 
 
 
 
 
 
 
 








function interview(round) {
  return new Promise((resolve, reject) => {
    if (Math.random() > 0.2) {
      resolve('success')
    } else {
      let err = new Error('cry')
      err.round = round
      reject(err)
    }
  })
}

function family(name) {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      if (Math.random() < 0.5) {
        const err = new Error('disagree')
        err.name = name
        reject(err)
      } else {
        resolve('agree')
      }
    }, Math.random() * 400)
  })
}


interview(1)
  .then(res => {
    return interview(2)
  })
  .then(res => {
    return interview(3)
  })
  .then(res => {
    // 多任务并发，只有family全部通过，才会进入 then 回调
    return Promise.all([
      family('father').catch(() => { }),
      family('mother'),
      family('sister')
    ]).catch(e => {
      e.round = 'family'
      throw e;
    })
  })
  .then((res) => {
    console.log('success')
  })
  .catch(err => {
    console.log('cry at', err.round, 'round')
  })

# Async-Await

在业界有一个异步流程的终极解决方案： Async-Await , 下面讲解一下它的要点：

async 其实就是 Promise 的语法糖，当它修饰一个 function 时，就相当于一个返回了 Pormise 对象的普通函数，而函数体的返回值分为三种情况：

  • 如果通过 return 返回了数据，就相当于返回 Promise 的普通函数调用了 resolve
  • 如果通过 throw 抛出了异常，就相当于普通函数调用了 reject
  • 如果没有写 return 语句，就相当于默认调用了 resolve 并且没有传参数

await 的功能就比较强大了，它可以将异步变为同步，它的作用如下：

  1. await 可以让 async 这个函数停下来，直到 await 修饰的异步函数执行完毕，再执行 async 后续的逻辑，相当于把异步变成同步
  2. 由 await 抛出的异常可以被 try catch 捕获到
