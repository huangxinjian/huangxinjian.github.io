# Async Await 知识点详解

Async-Await 被称为异步调用流程控制的终极解决方案，下面就来详细讲解一下这个知识点

# Async

Async 其实就是 Promise 的一个语法糖，例如以下两种写法的效果是一模一样的:

function f() {
    return Promise.resolve('TEST');
}

// 等价于上面的 Promise 函数
async function asyncF() {
    return 'TEST';
}

# Await

Await 关键字，它只能在 Async 函数内使用，让我们可以等待一个 promise。因此它最大的功能其实是，代替 .then 方法，将异步的逻辑变成同步的代码。

例如以下两个代码片段，对比一下使用 promise 和使用 await 的处理串行异步任务的区别：

// 使用 .then 进行串联操作
function asyncFunc() {
  otherAsyncFunc1()
  .then(function(x){
    return otherAsyncFunc2();
  })
  .then(function(x) {
    console.log(x)
  })
}

// 使用await关键字
async function asyncFunc() {
    const result1 = await otherAsyncFunc1();
    const result2 = await otherAsyncFunc2();
}

对于多个并行异步任务，使用 await 结合 promise.all 更加简洁：

// 使用 .then 方法
function fn1() {
    let p1 = rp('http://www.baidu.com');
    let p2 = rp('http://www.baidu.com');
    Promise.all([p1, p2])
        .then(function([res1, res2]) {
            console.log(res1,res2)
        })
}

// 使用await 关键字
async function fn1() {
    let p1 = rp('http://www.baidu.com');
    let p2 = rp('http://www.baidu.com');
    let [res1, res2] = await Promise.all([p1, p2]);
    console.log(res1,res2)
}

# Await 的实质

Await 其实也是一颗语法糖而已，它相当于 Promise.then() 的语法糖，对于所有写在 Await 后面的代码，在解析过程中都会被封装到 .then 函数体中，例如：

// await写法
await foo();         
console.log("hello");

// .then写法
foo().then(() => {
    console.log("hello");
});

# Await 错误处理

既然 Await 是 Promise.then() 的语法糖，那如果发生了异常可咋整咧，这里就需要使用 Await 的另一个功能了，在 Await 后续代码抛出的异常中，可以被 try catch 捕获。

(async () => {
  const fetchData = () => {
      return new Promise((resolve, reject) => {
          setTimeout(() => {
              // 抛出异常
              reject('fetch data is me')
          }, 1000)
      })
  }

  try {
    let result = await fetchData()
    console.log(result)
  } catch (error) {
    console.log(error) //fetch data is me
  }
})()
