# 异步调用

在 NodeJS 中，存在非常多的异步函数，这些函数的执行效率十分高效，也是 NodeJS 之所以拥有如此高性能的原因。

# 异步和同步

有异步，就必然有同步，那两者之间到底有什么区别呢？ 其实区别很简单，例如我向教务处打电话想要查询成绩，然后教务处说马上就查，叫我先别挂电话，查到之后马上告诉我。那么，在查成绩的这段时间里，我们什么都做不了，只能像个憨憨一样拿着电话等教务处给答复。这样就称之为同步。 与之相反，异步就是在我打电话给教务处后，教务处说记下我的电话，等查到成绩后就通知我，那么，在他查成绩的这段时间里，我可以做任何的事情直到他来通知我成绩是多少。

所以，异步和同步的区别就是：

  • 异步：在发出请求到获取结果之间的这段时间里，可以去做别的事情
  • 同步：从发出请求后，直到获取到结果之前，都只能等待，无法去做别的事情

# 浏览器和node的异步原理

在异步调用当中，我们自己是无法独自完成异步，必须依靠某个东西和我们共同完成，就好比上个例子当中的教务处负责人，他负责接收我的查成绩的请求，并在查到成绩后主动给我电话或发短信通知我。

所以不管什么异步模式当中是一定有这样一个核心中间人，去做最重要的三件事：接收异步请求、发生异步交互、返回异步响应

# 浏览器中的中间人

我们知道在浏览器当中异步模式是依赖于Ajax的，而Ajax中的核心，或者核心中间人就是XMLHttpRequest，如下图： 如上图,Ajax定义好请求和回调函数后，剩下的事情就交给XMLHttpRequest处理，XMLHttpRequest会和服务器交互，并产生时间差役，所以异步操作能够很好的解决这个问题，不需要刷新页面就能获取数据。

# NodeJS中的中间人

我们知道在Node当中异步模式是依赖于异步非阻塞I/O模型的，而这种模型中的核心，或者核心中间人就是EventLoop，如下图： 如上图，我们在调用Node API方法的时候，会把具体操作和回调函数交给EventLoop去执行，EventLoop维护了一个事件队列，当异步函数执行的时候，会把回调函数放进事件队列当中，V8引擎知道异步函数执行完成才会开始处理EventLoop。

这意味着JavaScript代码不是多线程的，及时好像看起来能同时执行多个线程的任务

# 异步存在的问题

由于这些函数的执行结果的返回时间是不确定的，因此异步存在一个很大的问题就是，我们在某一个时刻获取到的结果不一定是我们想要的，例如：

var result = 2
setTimeout(()=>{
  result = 3
}, 500)
console.log(result)

# NodeJS中的异步方式

在 NodeJS 中，提供了两种自带的异步写法：

  • Callback
  • EventEmitter

# Callback

Callback 是很常见的一种异步写法，在 NodeJS 中，对于 Callback 是有一定的规范的，这个规范通常称为：

  • Error First Callback
  • Node Style Callback

意思就是，对于所有的回调函数，第一个参数都统一返回 Error，后面的参数再开始返回值，例如：




 

 












function interview(callback) {
    if (Math.random() > 0.5) {
        // 如果成功，第一个参数给 NULL
        callback(null, 'success') 
    } else {
        callback(new Error('cry'))
    }
}

interview((err, res) => {
    // 在回调中，直接判断第一个参数是否存在，即可知道成功与否
    if (err) {
        return console.log('cry 1nd')
    }
    return console.log(res)
})

# EventEmitter

这是 Node API 中提供的方式，事实上这是观察者模式的实现。订阅者通过监听一些自定义事件，发送者无需关心是否有人在监听该事件，只需负责将消息往该事件发送

类似于 redis 中的发布订阅

使用方法也十分简单，首先定义一个发布者模块














 









const EventEmitter = require('events').EventEmitter

// 观察者模式
/**
 * 通知如何通过调用获取，何时通过观察者获取
 * 1. 如果发消息的人不需要知道是否有人在看消息并回复，例如app推送，就可以观察者
 * 2. 否则建议使用调用方式获取
 */
class GeekTime extends EventEmitter {
    constructor() {
        super()
        // 每两秒输出一个事件
        setInterval(() => {
            this.emit('newLession', {'price': Math.random() * 100})
        }, 2000)
    }
}

const geek = new GeekTime()

// 暴露一个单例对象出去
module.exports = geek
