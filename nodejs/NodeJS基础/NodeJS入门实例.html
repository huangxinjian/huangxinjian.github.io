# Hellp CommonJS

Node.js 是基于 CommonJS 规范的实现，每个文件都是一个模块，并且内容需要严格遵守 CommonJS 的规范。

同时也是 <scripts> 标签在 NodeJs 中的一个解决方案

我们平时写的项目，一般都会有数十个文件，即数十个模块，多个文件调用的核心都是 基于模块对外暴露接口和相互引用。

​

下面介绍一下 Node.js 里的 CommonJS 的写法，主要涉及到两个文件：

  1. hello_commonjs.js
  2. hello_commonjs_test.js

# 创建模块

首先通过 module.exports 对外暴露一个匿名函数，在函数中打印 Hello CommonJS :

module.exports = function(){
    console.log('Hello CommonJS!');
}

# Hello HTTP

Node.js 有一个最大的亮点之一就是创建 Web 应用 ，仅仅需要几行代码，就可以创建一个 Web服务器 ，而不需要类似 java 那样需要另外的 Tomcat 容器。

下面我们来创建一个 web 容器：

var http = require('http');

http.createServer(function (req, res){
	res.writeHead(200, {'Content-Type': 'text/plain'});
	res.end('Hello Node.js\n');
}).listen(3000,"127.0.0.1");

console.log('Server running at http://127.0.0.1:3000/');

关于上面的代码，有几点注意事项：

  1. http 是 Node.js SDK 里的内置模块，也就是 NodeBindings 实现的 JavaScript API 编写的 Node.js标准库。
  2. 通过 http.createServer 创建了一个 HTTP 模块。
  3. 通过 listen 方法指定端口和 IP 地址，启动服务。
  4. res 是响应（response）的意思，通过该对象控制对浏览器的操作，设置返回状态码是 200，设置 HTTP 头字段里的Content-Type 为 test/plain ，最后向浏览器返回 Hello Node.js 文本，由浏览器解析并渲染。

# 切换 V4 版本

Node.js V4.x 里，如果不使用严格模式，在块作用域内是无法使用 let、const、function、class这些关键字的。只有开启了严格模式，才能使用这些关键字，并非 Node.js V4.x 不支持。
