# Node.js 安装

Node.js 主要通过 3m 安装来管理各个模块

  1. nvm （node version manager）：用于开发阶段，解决多个Node.js版本共存、切换、测试等问题
  2. npm（node package manager）: 解决 Node.js 模块安装问题
  3. nrm（node registry manager）：解决 npm 镜像访问慢的问题，提供测速、切换下载源功能

Node.js 支持 macOS 、 linux 、window 等多个主流系统，但在 window 平台下可能会出现一些莫名其妙的问题。

# nvm

nvm 的主要作用是管理多个 Node.js 的版本 ，下载地址

另外，nvm 当前并不支持 window 操作系统，如果需要在 window 下使用包管理器，则需要使用 nvm-window 或者 nodlist

同时还有一款跨平台的包管理工具：nvs

这是将来安装 Node.js 放置的路径，切换版本的时候，会将要切换的版本的文件替换到这里。

# 安装指定版本的 Node.js 👍

注意，由于大陆的服务器访问 Node.js 的默认下载地址 ： https://nodejs.org/dist 会十分缓慢，因为需要切换到国内的镜像源。

通过命令 nvm node_mirror https://npm.taobao.org/mirrors/node/ 切换 node.js 镜像源

通过命令 nvm npm_mirror https://npm.taobao.org/mirrors/npm/ 切换 npm 镜像源

如果想要安装指定版本的 Node.js ，只需要使用以下命令即可 ： nvm install <version> [arch] ，其中

  1. version 是 Node.js 的版本号，推荐选择 LTS 版本安装，因为该版本是官方长期支持版本，而 Current 是尝鲜版，一般用于开发者学习。
  2. arch 是操作系统的位数，默认是 64位，如果要安装32位的，需要指定出来，如 npm isntall 10.16.0 32

# 使用指定版本的 Node.js

执行命令 nvm use <version> [<arch>] 来进行版本间的切换

# npm

npm 的作用是 Node.js 包管理器，主要功能是管理 Node.js 包，包括安装、卸载、更新、查看、搜索、发布等

包就是 NodeJs 模块

npm 是随着 Node.js 一起安装的包管理工具，并且其本身也是一个 Node.js 模块。主要的应用场景如下：

  1. 从 npm 镜像服务器下载第三方模块
  2. 从 npm 镜像服务器下载并安装命令行程序到本地（vue）
  3. 自己发布模块到 npm 镜像服务器供他人使用

# npm 各版本的区别

  1. npm v2 : 树形依赖、层次深、依赖重叠、安装时间长，但相对更清晰

  2. npm v3 : 扁平化依赖

  3. npm v5 : 自动记录依赖树，重写了缓存系统，下载的时候使用了强校验

重要特征是，新增了 package-lock.json 文件。 在操作依赖的时候默认生成，用于记录和锁定依赖树的信息

# 升级 npm 👍

由于 npm 本身也是一个Node.js模块，因此只需要执行 npm install -g npm@version ，在 version 处指定安装的版本号即可。

安装完成后，需要指定 npm 的下载镜像地址为：

npm config set registry https://registry.npm.taobao.org
1

# 使用 npm 安装模块

可以通过 npm -h install 查看帮助命令

在npm 中安装模块一共分为两大类：

  1. 本地安装（Local）
  2. 全局安装（Global）

两者的差别仅仅在于安装时命令行是否存在 -g 指令

常用指令选项：

命令 简写 说明
无选项 无 将模块安装到本地的 node_module 目录下，但是不会记录到 package.json
--save-prod -P 将模块安装到本地的 node_module 目录下，同时保存到 package.json 的 dependencies ，这在安装模块的时候是必须安装的
--save-dev -D 将模块安装到本地的 node_module 目录下，同时保存到 package.json 的 devDependencies，仅供开发时使用，一般测试模块等辅助安装工具均这样安装
--global -g 安装的模块为全局模块，如果是命令行模块，会自动添加到环境变量

# 何为本地安装

关于本地安装，有如下三点说明：

  1. 本地安装的模块均安装到 node_modules 下，即 运行 npm 命令时所在的目录 下。
  2. 如果该目录下不存在 node_modules 文件夹，会被自动创建。
  3. 可以通过 require() 引入本地安装的包。

# nrm

nrm 是用来管理多个镜像源的地址，它可以帮我们简单、快速地在多个源直接进行切换，并且默认内置了 npm、cnpm、taobao、nj、rednpm、npmMirror 等镜像源

​ 我们国内的用户在下载一些 Node.js模块 的时候，因为 npm官方源 是国外的站点，访问速度会比较慢，因此我们都会切换我们的下载站位为国内的一些镜像源。

​ 例如设置为淘宝的镜像源：

npm config set registry https://npm.taobao.org/mirrors/npm/
1

​

​ 但是有时候我们可能也会切换到公司内部的镜像源，在多个镜像源直接需要切换的时候，每次都通过手动设置就会十分的麻烦，这时候就需要 nrm 来替我们管理多个源。

​

# 安装 nrm

nrm 是一个命令行模块，需要使用 --global 全局安装

nrm 其实也是 Node.js 的一个模块，因此想要安装 nrm ，就跟使用 npm 安装模块是一样的，命令如下：

npm install --global nrm
1

# 测速

​ 下载成功后可以通过 nrm test 来测试各个镜像源的速度，其中 * 代表当前正在使用的源

$ nrm test
