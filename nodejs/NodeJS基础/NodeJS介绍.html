# Node.js 是什么？

# 什么是 Node.js ❓

Node.js 诞生于 2009 年，是由 Ryan Dahl 开发完成的，号称编程界的版本帝， 目前发展到最新的版本已经是 v12.7.0 ，而最新的长期支持（LTS）版本是 v10.16.0。官方网站 github

那什么是 Node.js 呢？ 在 Node.js官网 上对其介绍的总结如下：

  1. 它既不是一门语言，也不是一门框架，它是一种 基于 Chrome V8 引擎的 JavaScript 运行时环境。

  2. Node.js 构建在 **Chrome V8 ** 这个 JavaScript引擎之上。

而该引擎是通过 C/C++ 编写的，因此在运行时相当于要将 JavaScript 转化成底层的 C/C++ 语言后再执行。

  1. Node.js 的每个函数都是同步的，而 I/O操作是异步的。

所有由 JavaScript 编写的 I/O 操作，最终都会通过 libuv一款由 C/C++ 编写的**事件循环(Event Loop)**处理库来处理。

  1. Node.js 使用 npm 作为包管理器，该管理器是目前开源库里最大的包管理生态，功能十分强大。

简单的来说，

  • JavaScript 让我们可以控制浏览器
  • NodeJs 让我们可以控制计算机

# Node.js 架构图

Node.js 是基于 Chrome V8 引擎构建的，由事件驱动 Event Loop 分发 I/O任务，最终工作线程 Work Thread 会将任务放到线程池 Thread Pool 中执行，事件驱动只需要等待执行的结果即可。

Node.js 的早期架构图如下:

  1. Chrome V8 引擎 ： Google 开源的一款由 C/C++ 编写的 JavaScript 引擎

  2. Node.js 内置 V8 引擎，所以使用的是 JavaScript 语法。

  3. JavaScript 语言的一大特点是单线程，同一时间只能做一件事，因此多个任务执行则需要排序等候

  4. 等待中的 I/O任务 会被放到 Event Loop 中，该功能由 libuv 提供

  5. Event Loop 负责将文件 I/O 任务放到线程池中，线程池由 libuv 提供

  6. 网络 I/O 任务不通过线程池处理

# Node.js 原理图

下面换个角度看一下 Node.js 的原理：

首先是最底层的组成：

  1. Chrome V8 引擎负责解析并执行 JavaScript (这就是为什么浏览器能够执行 JavaScript 的原因)
  2. libuv 由线程池和事件循环组成，负责所有 I/O任务 的分发与执行
  3. Node.js 在接受任务的时候是单线程的，但是在执行任务的时候是多线程的。

然后是 Node Bindings :

  1. 将 Chrome V8 引擎暴露的 C/C++ 接口转化为 JavaScript API
  2. 并通过这些 API 编写 Node Standard Library
  3. 所有的这些 API 统称为 Node.js SDK

# Node.js 的特点

在 《狼书（卷1）：更了不起的Node.js》 书中有这么一句话：

Node.js 是可扩展的适合用于构建高性能Web应用的最简单的解决方案。

因此其特点如下：

  1. 适合构建 Web 应用
    1. 构建网站，单体式应用
    2. 构建 API 供不同的应用调用
    3. 构建 RPC（Remote Precedure Call 远程服务调用） 协议服务，即将数据库访问返回的数据，以 TCP 的形式传输给调用方。
    4. 前后端分离
    5. 适用于 Serverless , 无需再关心运维
  2. 高性能
    1. Node.js 是构建在 Chrome V8 引擎上执行的，性能非常快
    2. Node.js 每个 I/O 任务都是异步的，统统继承到 libuv 中进行处理。
    3. 适用于 I/O 密集的网络应用开发
  3. 可扩展
    1. 可以轻松搭配 Java 语言使用
    2. 可以使用 npm 上的大量模块
    3. 架构互补
  4. 简单
