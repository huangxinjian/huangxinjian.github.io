# Kafka 集群搭建

以下环境均在 vmware centerOS7 中搭建

下面详细讲解一下 Kafka 的集群搭建，首先我们需要下载 Kakfa，由于国内的网速比较慢，推荐使用清华大学的镜像：

wget https://mirrors.tuna.tsinghua.edu.cn/apache/kafka/2.2.1/kafka_2.12-2.2.1.tgz
1

# 安装并配置第一台主机

下载完成后，我们进行解压缩

为了方便我们使用，我们可以为 kafka 创建一个不带版本号的软连接

解压完成后，我们需要修改 kafka 下的 config/server.properties 配置文件，其中需要修改的内容有：

# kafka 服务器id
broker.id = 0

# kafka 对外提供的端口, socket 服务器会监听该端口
listener=PLAINTEXT://192.168.59.122:9092

# 设置 zookeeper 的地址
zookeeper.connect=192.168.59.117:2181

# Kafka 的启动与停止

首先要明确，对于 kafka 的一些命令，我们并不需要记住，这些在官方文档上都可以查阅：官方地址

# 启动 Zookeeper

在 Kafka 介绍 中，我们提到了 Zookeeper 对于 kafka 的作用，可以认为，kafka 其实就是 Zookeeper 的客户端。

因此，我们在启动 kafka 之前，需要先启动 zookeeper, 如果是使用 kafka 内置的 zookeeper，可以这样启动

bin/zookeeper-server-start.sh config/zookeeper.properties
1

如果是自己另外配置了一台 zookeeper，就去到那台机器下启动：

zkServer.sh start
1

# 启动 Kafka

然后我们启动 Kafka Server

bin/kafka-server-start.sh config/server.properties
1

注意这样启动有个问题，就是我们当前的命令行窗口会进入阻塞状态，无法再输入别的命令

所以我们需要让 kafka 以守护进程的方式启动，需要添加参数 -daemon ：

bin/kafka-server-start.sh -daemon config/server.properties
1

启动后可以通过 ps 命令查看 kafka 进程是否存在：

ps aux | grep kafka
1

# 停止 Kafka

如果需要停止 Kafka，可执行以下脚本

bin/kafka-server-stop.sh
1

# Kafka 操作

首先要明确，对于 kafka 的一些命令，我们并不需要记住，这些在官方文档上都可以查阅：官方地址

# 创建 topic

下面创建一个名为 test 的 topic

bin/kafka-topics.sh --create --bootstrap-server localhost:9092 --replication-factor 2 --partitions 3 --topic country
1

其中：

  • --bootstrap-server 代表我们要将请求发给哪台 kafka 服务器处理，因为需要将 localhost 替换为对应的 IP 地址
  • --partitions 代表这个 topic 有多少个分区
  • --Replication-factor 代表这个 topic 的每个分区有多少个副本

例如我们这里创建了 3 个分区，就会在三台机器中的 /tmp/kafka-logs 文件夹中随机生成：

  • country-1
  • country-2
  • country-3

这三个文件夹，这个文件夹就是 partition 分区。

然后我们设置了 --partitions 2 ，就是说每个分区都会存在一个副本，并且两个相同分区之间存在 leader 和 follower 的关系。

另外，这些副本存在哪一台机器上也是随机的。有可能机器 A 是 country-1 和 country-0 ，机器 B 是 country-1 和 country-2，机器 C 是 country-0、country-2。 这些都是由 Zookeeper 进行配对的。

# 删除 topic

如果我们需要删除某个 topic ，可以执行这条命令

bin/kafka-topics.sh --delete --bootstrap-server 192.168.59.122:9092 --topic country

# 查看分区配对信息

前面说到，每个分区会随机分配给集群中的机器，如果存在副本集，也是随机分配的，并且同一分区中的多个集合存在 Leader - Follower 关系。

对此，我们可以通过以下命令进行查询：

> bin/kafka-topics.sh --describe --bootstrap-server 192.168.59.122:9092 --topic country

# 发送消息

创建好 Topic 后，我们就可以通过 生产者 来生产消息了。

创建生产者的命令如下：

bin/kafka-console-producer.sh --broker-list 192.168.59.121:9092 --topic country
1

然后就可以发送消息了

> bin/kafka-console-producer.sh --broker-list 192.168.59.121:9092 --topic country
This is a message
This is another message

# 消费消息

既然我们通过生产者生产出消息并发送到 Kafka 中，相对应我们也需要通过 消费者 来消费消息。

创建消费者的命令如下：

bin/kafka-console-consumer.sh --bootstrap-server localhost:9092 --topic country
1

执行完这条命令后，命令行窗口就会进入监听状态。此时生产者继续生产消息，消费者可以马上接受到。

如果需要消费一些在消费者创建之前，生产者就已经发送了的消息，可以通过添加选项 --from-beginning。例如：

> bin/kafka-console-consumer.sh --bootstrap-server localhost:9092 --topic country --from-beginning
This is a message
This is another message
