# Kafka API 使用教程

Kafka API 有两种比较常用，一种是原生 API，另一种是 Spring Kafka

# Kakfa 原生 API

注意，我们的虚拟机集群都需要启动了先，因为 API 只是作为生产者和消费者去连接我们的集群

想要使用 Kafka 原生API，首先需要引入依赖：

<!-- kafka 依赖 -->
<dependency>
    <groupId>org.apache.kafka</groupId>
    <artifactId>kafka_2.12</artifactId>
    <version>1.1.1</version>
</dependency>

# 不带回调的生产者

然后我们就可以创建我们的第一个生产者了, 在 Kafka 原生API中，KafkaProducer 代表生产者对象，我们需要为这个对象配置一些当初我们在使用命令时所配置的那些 option 信息，例如 --bootstrap-server

public class OneProducer {

  /**
    * 首先创建一个生产者
    * Interger 为 key 的类型
    * String  为 value 的类型
    */
  private KafkaProducer<Integer, String> producer;

  public OneProducer() {
    Properties properties = new Properties();
    // 这些常量在 ProducerConfig 中都有
    properties.put("bootstrap.servers", "kafkaOS1:9092,kafkaOS2:9092,kafkaOS3:9092");
    // 定义 key、value 序列化器
    properties.put("key.serializer", "org.apache.kafka.common.serialization.IntegerSerializer");
    properties.put("value.serializer", "org.apache.kafka.common.serialization.StringSerializer");
    this.producer = new KafkaProducer<Integer, String>(properties);
  }
}

创建好生产者对象后，我们就可以为它提供一个生产消息的方法 sendMsg 了：

/**
  * 发送消息
  */
public void sendMsg() {
    // 构造 Kafka 专用的消息对象

    // 方式①：提供 topic 和 消息内容 ，(String topic, V value)
    // ProducerRecord<Integer, String> msg = new ProducerRecord<>("cities", "zhaoQing");

    // 方式②：提供 topic、key 和 消息内容 ，(String topic, K key, V value)
    // ProducerRecord<Integer, String> msg = new ProducerRecord<>("cities", 0, "zhaoQing");

    // 方式③：提供 topic、哪个分区、key 和 消息内容 ，(String topic, Integer partition, K key, V value)
    // ProducerRecord<Integer, String> msg = new ProducerRecord<>("cities", 2, 0, "zhaoQing");

    for (int j = 0; j < 10; j++) {
      int k = j;
      ProducerRecord<Integer, String> msg = new ProducerRecord<>("cities", j * 10, "ee" + j);
      this.producer.send(msg);
    }
}

# 带回调的生产者

上面我们这种方式的发送消息，在发送之后，代码不会得到任何的回复，从而让我们不清楚，我们的消息是否已经发送成功了，因此可以对 producer 的 send 方法添加一个回调函数，用来确保我们的消息是否发送成功：

// 添加回调
this.producer.send(msg, ((recordMetadata, exception) -> {
    System.out.println("k =" + k);
    System.out.println("topic =" + recordMetadata.topic());
    System.out.println("partition =" + recordMetadata.partition());
    System.out.println("offset =" + recordMetadata.offset());
}));

# 批量发送的生产者

前面的 send 方法实质上是每执行一个 for 循环，就会向 broker 发送一次消息，这样的效率必然是很慢的，如果 kafka 只有这种效率，必然是无法支撑大数据的消息生产的。

前面在 Kafka 为何有这么高吞吐率的原因分析过，Kafka 之所以那么快，其中一个原因就是，Kakfa 支持批量发送。

想要开启批量发送，我们需要在生产者的构造方法中多添加两个配置：

// 批量提交，新增两个参数 16k和50ms
properties.put("batch.size", 10000);
properties.put("linger.ms", 15000);
1
2
3

其中 batch.size 设置的是 字节 byte 大小，一旦设置了这个值，生产者会先将消息缓存起来，直到消息达到这个字节大小，才会往 broker 中发送。

另外一个属性 linger.ms 则是设置当消费者上一次发送消息给 broker 后开始计时，当经过 linger.ms 后，不论消息大小够不够 batch.size 所设置的值，都会发送给服务器。

也就是说，两个条件，满足其一才会发送给服务器，这样就可以大大提高 kafka 的性能。

# 创建消费者

创建完生产者，下面我们创建一个消费者来消费消息，消费者需要配置的信息比较多，不过都可以在 ConsumerConfig 中查到：

public class OneConsumer extends ShutdownableThread {

  // 创建消费者对象
  private KafkaConsumer<Integer, String> consumer;

  public OneConsumer() {
    // 两个参数：
    // 1)指定当前消费者名称
    // 2)指定消费过程是否会被中断
    super("KafkaConsumerTest", false);

    Properties properties = new Properties();
    String brokers = "kafkaOS1:9092,kafkaOS2:9092,kafkaOS3:9092";
    // 指定kafka集群
    properties.put("bootstrap.servers", brokers);
    // 指定消费者组ID
    properties.put("group.id", "cityGroup1");

    // 开启自动提交，默认为true
    properties.put("enable.auto.commit", "true");
    // 指定自动提交的超时时限，默认5s
    properties.put("auto.commit.interval.ms", "1000");

    // 指定消费者被broker认定为挂掉的时限。若broker在此时间内未收到当前消费者发送的心跳，则broker
    // 认为消费者已经挂掉。默认为10s
    properties.put("session.timeout.ms", "30000");
    // 指定两次心跳的时间间隔，默认为3s，一般不要超过session.timeout.ms的 1/3
    properties.put("heartbeat.interval.ms", "10000");
    // 当kafka中没有指定offset初值时，或指定的offset不存在时，从这里读取offset的值。其取值的意义为：
    // earliest:指定 offset 为第一条offset
    // latest: 指定 offset 为最后一条offset
    properties.put("auto.off.reset", "earliest");
    // 指定key与value的反序列化器
    properties.put("key.deserializer",
            "org.apache.kafka.common.serialization.IntegerDeserializer");
    properties.put("value.deserializer",
            "org.apache.kafka.common.serialization.StringDeserializer");
    this.consumer = new KafkaConsumer<Integer, String>(properties);
  }
}

注意这里我们继承了 ShutDownableThread ，该类是一个可以循环执行某个方法（doWork方法）的线程，因此我们需要实现它的 doWork 方法

@Override
public void doWork() {
    // 订阅消费主题
    consumer.subscribe(Collections.singletonList("cities"));
    // timeout： 如果缓冲区中没有可用的数据，则在轮询中等待的时间(以毫秒为单位)。
    // 如果为0，则立即返回缓冲区中当前可用的所有记录，否则返回空值。
    // 不能是负数。
    ConsumerRecords<Integer, String> records = this.consumer.poll(1000);
    for (ConsumerRecord record: records) {
        System.out.println("topic = " + record.topic());
        System.out.println("partition = " + record.partition());
        System.out.println("key = " + record.key());
        System.out.println("value = " + record.value());
    }
}
