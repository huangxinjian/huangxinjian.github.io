# 接口和Lambda和内部类

接口、Lambda表达式和内部类，都是面向对象中的高级知识，下面将详细介绍它们。

# 接口

接口，又称为 interface, 主要用来描述一个类具有什么功能，但无需给出每个功能的具体实现。换句话说，接口只需要定义方法签名，无需给出方法体。

接口不是类，只是一个用来描述类的一组需求，这些类需要遵循接口描述的格式统一定义。

# 定义和实现

定义一个接口很简单，只需要通过关键字 interface 即可

public interface MyInterface {
  void sayHello(String name);
}
1
2
3

这样就可以定义一个 MyInterface 接口，并描述了一个 sayHello 方法，如果有类需要实现它，也需要实现这个方法。实现接口通过关键字 implements 指定，例如：

public interface MyInterfaceImpl implements MyInterface {
  public void sayHello(String name) {
    System.out.println("hello" + name);
  }
}

# 静态方法

在 JDK 1.8 之后，新增了一个特性，那就是在接口中允许定义静态方法。例如：

 public interface MyInterface {
   int age = 10;
   void sayHello(String name);
   static void sayHi() {
     System.out.println("hi");
   }
 }

# 默认方法

在 JDK 1.8 之后，还可以定义默认方法，只需要通过 default 标注即可。

public interface Comparable<T> {
  default int compareTo(T other) { return 0;}
}
1
2
3

默认方法的好处是，当实现接口的时候，不需要实现接口内的全部方法，则只需要根据自己需要的方法去实现即可。

# 函数式接口

对于只有一个抽象方法声明的接口，称之为函数式接口，这里要注意这个抽象的意思，前面提到，JDK8 以后接口可以定义静态方法和默认方法，但并不意味着如果在接口中新增了这些方法，该接口就不是函数式接口。

例如：

public interface A {
  void sayBye();

  default void sayHello() {
    System.out.println('hi');
  }
}

在这里的接口 A 仍然是函数式接口，即使定义了默认方法

为了防止程序员不小心在函数式接口中定义了多个抽象方法，可以通过注解 @FunctionalInterface 标注在类上，例如：

@FunctionalInterface
public interface A {
  void sayBye();
}

# 接口的特点

  1. 接口不是类，不能通过 new 创建实例
    x = new MyInterface(); // ERROR !
    1
  2. 接口中不能包含实例域，但是可以包含常量，因为会被自动修饰 public static final.
    public interface MyInterface {
 int age = 10;
 void sayHello(String name);
}
    1
    2
    3
    4
  3. 接口中的所有方法都会被自动修饰为 public abstract
  4. 接口支持多继承，类不支持。
  5. JDK1.8之后，接口允许新增静态方法和默认方法
